Hexo搭建个人博客（一）

Posted on 2018-02-08

1、前言

现在很多网站提供的个人博客充满了各种广告，写起来也不方便，比如程序代码无法高亮等，看起来一点都不爽，找了个时间研究了下目前常用的独立博客的做法，比较深受大家喜爱的是通过hexo来生成静态网页，通过github提供的空间及二级域名进行免费发布。

本人是初次玩博客，手头也有vps主机，目前打算这样：

  • 第一步：单台win7上实用Hexo+Github静态网页托管
  • 第二步：多终端（家里、单位）win7上更新博
  • 第三步：将Hexo博客架设到VPS上，同时支持多台电脑更新博文

2、单台win7上实现Hexo+Github静态网页托管

hexo是用nodejs写的一个简洁、高效的博客框架，通过md可以快速生成静态网页，其主题有很多种可供选择。hexo是免费开源的（感谢共享精神）。

hexo.io中文手册：https://hexo.io/zh-cn/docs/

2.1 准备环境

2.11 github上申请账号

github用于提供开源代码托管，提供个人、项目两种方式的仓库，免费300M空间，在线仓库通过git可实现团队操作。国内用户通过科学上网来访问。

申请主页：https://github.com/

2.12 下载Git、Node.js

  1. Git下载：

https://git-scm.com/download/win //本人下载的是(32-bit Git for Windows Setup)

https://github.com/waylau/git-for-win //国内下载，版本滞后

  1. Node.js下载：

https://nodejs.org/en/ //本人下载的最新版node-v9.5.0-x86.msi

2.13 安装Git、Node.js

  1. Git安装

双击安装文件，默认next结束，右击桌面选择“Git Bash Here”，进入Git，输入git --verison验证是否成功安装

  1. Node.js安装

node.js可以通过nvm安装，对于windows用户来说，建议使用安装程序进行安装。双击node-v9.5.0-x86.msi保持默认一路next安装结束。打开cmd输入node -v验证是否成功安装。

2.2 安装Hexo

2.2.1 Git账户设置

git config --global user.name "xxx"及email "xx@xx.com"用于设置每次提交的作者信息，包括姓名和邮箱，–global表示对所有项目都使用这个信息。git config --list可查看设置。

2.2.2 安装hexo

  1. 切换npm源

    默认是通过npm来安装hexo-cli，在使用npm时，官方的源下载npm包会比较慢，国内一般使用淘宝npm镜像源，切换npm源过程如下：

  1. 安装hexo

    通过 npm install -g hexo-cli进行安装

  1. hexo初始化

hexo初始化是释放hexo相关文件，本人博客目录是e:/blog，因此需切换至该目录，输入hexo init

  1. 最后输入npm install

2.5 发布到github

大体流程是本地hexo将写好的博文（MD文件），通过hexo g来生成public，然后通过hexo d将public文件内容推送到github，其他人即可用github二级域名来访问生成的静态网页。

2.5.1 github建立仓库

  1. 新建一个仓库

  1. 注意个人仓库的名字必须是username/username.github.io，后续可通过https://username.github.io

来访问你的个人主页，我的就是https://cheneytac.github.io，这里已经创建了，点create repository确定。

2.5.2 添加ssh访问权限

本地客户端须通过ssh方式与远端的github建立联系，远端github作为一个仓库来接受本地客户端内容的推送。因此，本地客户端要生成ssh公钥及私钥，再将公钥传到github上。

  1. 用ssh-keygen -t rsa -C "cheneytac@qq.com"生成本地公私钥
  1. 复制.ssh下的id_rsa.pub公钥复制到github

输入之前创建的密码后添加。

  1. 测试ssh访问githubssh -Tgit@github.com

  1. 新建博文

到E:\blog\source_posts下找到xxx.md，然后使用你喜欢的md软件进行书写。

3、相关报错处理

按照以上步骤step by setp来建立，正常不会报错，用hexo写博文正确姿态应是：建立md文件–>书写修饰博文–>查看本地预览–>发布到github。在此过程中有可能会遇到以下问题：

  • 修改参数或博文后，本地hexo s即报错
  • 无法同步情况

第一种情况很多都是因为语法后缺少空格，也有这种：
